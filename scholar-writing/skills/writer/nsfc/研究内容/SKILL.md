# 研究内容 Writer

## 任务

撰写国自然申报书的「研究内容」部分。这是全文回答"做什么"的核心章节。

你将收到：
1. 该章节的大纲（来自 planning/outline.md）
2. 该章节的评审标准 checklist（写作时请对照）
3. 已完成前序章节的摘要（200-300 字/章）
4. 相关素材片段（来自 planning/material_mapping.md）
5. 论点注册表（planning/claim_registry.md）
6. 立项依据全文（本章必须与其严格对应）

## 写作策略

### 两阶段写作法

**阶段1：规划（输出骨架）**

产出结构骨架，包含：
- 研究目标（3个以内）
- 各子内容的标题、核心问题、方法关键词
- 子内容间的逻辑递进关系
- 内部编号（S1-S4，仅阶段1使用，最终输出清除）

骨架输出后，在 `<!-- STAGE1_SKELETON -->` 标签中，供用户确认。如果 pipeline 处于自动模式，直接进入阶段2。

**阶段2：叙事（填充全文）**

在确认的骨架上填充完整叙事，保持递进感和前后关联。

### 推荐章节结构

#### 1. 研究目标（5-8行）

- 3个以内目标，用序号标出
- 用大同行能够理解的术语描述
- 每个目标使用精确动词（明确、揭示、验证、建立、阐明）

目标动词选择：
| 层次 | 动词 | 适用场景 |
|------|------|---------|
| 描述性 | 明确、确定、鉴定 | 现象/关系的确认 |
| 解释性 | 揭示、阐明、解析 | 机制/规律的探索 |
| 验证性 | 验证、证实、检验 | 假说/模型的验证 |
| 建设性 | 建立、构建、发展 | 方法/模型的创建 |

#### 2. 总体思路（1-2段 + 框架图描述）

- 描述各研究方向的整体关系和递进逻辑
- 建议采用三层架构描述（理论层→方法层→应用层）
- 说明层间关系（"相互支撑、逐级递进"）
- 文字描述研究内容框架图的结构（供后续制图参考）

#### 3. 各研究内容（3-4个子内容）

每个子内容遵循7步通用结构：

1. **小标题**：`研究内容X：***的***研究`
2. **主题定义句**（可选，工程/信息类可用比喻式定位）
3. **提出问题**：该子内容要回答的具体问题（1-2句）
4. **研究方案**：拟采用的方法/实验设计（3-5句）
5. **关键技术**：如有创新方法需详述
6. **可行性融入**：1-2句说明方法的可行性依据
7. **预期结果**：可量化的目标和判断标准（1-2句）

**关键对应关系**：每个研究内容必须对应立项依据中识别的一个研究空白/挑战。

#### 4. 拟解决的关键科学问题

论证模式：`现状描述 → 现有方法不足 → "因此，如何..." → "是本项目拟解决的第X个关键科学问题"`

多个问题应形成递进链条。

#### 5. 条目间关系（1段）

说明3-4个研究内容之间的逻辑递进关系：
- **层次1（基础）**：解决核心理论或方法问题
- **层次2（核心）**：在层次1基础上构建核心技术
- **层次3（应用/验证）**：将核心技术应用于具体场景并验证

各层次之间的输入输出关系要明确。

### 内部编号管理

写作过程中可使用内部编号（S1-S4）标记子内容，方便与大纲和依赖图对照。最终输出时清除所有内部编号，只保留正式标题。

### 字数与篇幅

- 面上项目：10000-14000字（6-8页）
- 青年基金：6000-8000字（3-5页）
- 研究内容条目数：面上 3-5个，青年 2-3个

### 一致性检查

写作完成后自检：
- 每个研究内容是否都在回答立项依据提出的科学问题
- 研究内容之间是否有逻辑关系（递进、互补、并行验证）
- 是否有研究内容与科学问题无关
- 术语表述是否与 claim_registry 一致
