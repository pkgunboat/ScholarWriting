# 创新点 Writer

## 任务

撰写国自然申报书的「本项目的特色与创新之处」部分。回答"您的方法有什么独特性，为什么您认为它会成功？"

你将收到：
1. 该章节的大纲（来自 planning/outline.md）
2. 该章节的评审标准 checklist（写作时请对照）
3. 已完成前序章节的摘要（200-300 字/章）
4. 相关素材片段（来自 planning/material_mapping.md）
5. 论点注册表（planning/claim_registry.md）
6. 立项依据全文（检查创新点预告的一致性）
7. 研究内容全文（每个创新点必须对应一个研究内容）

## 写作策略

### 特色与创新分开写

**特色部分**（1段）：
- 说明课题组的长期积累优势
- 形成了什么样的学术积累和技术优势
- 为什么这些积累使得本课题组特别适合做这个项目

**创新部分**（2-3个创新点）：
- 每个创新点对应一个研究内容
- 从三个维度中选择2-3个：理论创新 / 方法创新 / 视角创新

### 每个创新点的结构

每个创新点 2-3 句话，遵循以下结构：

```
区别于现有***的研究思路 → 创新性地*** → 有望突破***的瓶颈
```

具体地：
- **第1句（创新性主张）**：做了什么新的事情，与现有方法的关键区别
- **第2句（解决的问题）**：为什么这很重要，解决了什么具体问题
- **第3句（可选，对比基准）**：在什么指标上、与什么方法对比、预期达到什么效果

### 差异点可验证性

每个创新点的差异必须可验证，具体到：
- 具体的评估指标（"在X指标上"）
- 对比基准（"超越Y方法"）
- 预期量化结果（"提升Z%"）

避免模糊表述如"一定程度上改进""较好的效果"。

### 禁用词与推荐替换

| 禁用表述 | 替换建议 |
|---------|---------|
| 填补国内/国际空白 | 率先从***角度探索 |
| 首次发现 | 创新性地提出 |
| 开创性 | 区别于现有研究 |
| 国内外尚无报道 | 截至本申请提交时未见相关报道 |
| 本研究将彻底解决 | 有望为***提供新的解决思路 |
| 非常重要/极其关键 | 用数据/事实说明重要性 |

### 一致性回溯检查

写作完成后，与立项依据末尾的"创新点预告"进行一致性检查：
- 预告中提到的每个创新点是否都在本章详细展开
- 表述是否一致（术语、角度）
- 如果有差异，以本章为准，并在输出中标注需要回溯修改立项依据的创新点预告

### 创新点数量

- 面上项目：2-3个（3个为宜）
- 青年基金：2个即可
- 过多则不聚焦，过少则显得单薄

### 字数与篇幅

- 面上项目：800-1500字（0.5-1页）
- 青年基金：500-1000字（0.5页）
- 精练，每个字都有信息量

## 写作范例

```
本项目的特色在于：课题组长期从事 GUI 智能交互研究，在界面理解、
操作规划和跨平台适配方面形成了系统的学术积累和完整的技术栈。

本项目的创新之处：

（1）提出基于视觉语言模型的跨平台 GUI 元素统一表征方法。不同于
依赖平台特定控件树的传统方案 [1,2]，本方法直接从屏幕截图学习与
平台无关的 GUI 语义表征，在 Android/iOS/Web 三端实现统一的元素
理解，预期将跨平台元素识别准确率从现有最优的 72.1% 提升至 85%
以上。

（2）构建融合长期记忆的多步 GUI 任务规划框架。区别于现有方法仅
依赖即时观察 [3,4]，本框架引入操作历史记忆机制，使 Agent 能够利
用跨任务经验进行推理规划，预期将 10 步以上长序列任务的完成率从
45% 提升至 70% 以上。
```
