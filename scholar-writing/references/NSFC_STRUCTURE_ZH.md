# 国自然申报书章节结构规范

本文档定义各章节"该写什么、怎么组织"的结构蓝图，供 Writer Agent 在写作时作为结构参考。与 `SENTENCE_PATTERNS_ZH.md`（通用句式零件）互补：本文档回答"结构层"问题，句式库回答"措辞层"问题。

---

## 1. 写前自检：基金委"灵魂六问"

撰写前务必自问，贯穿全文：

1. **该项目想做什么？** 用大同行能够理解的术语表达研究目标。
2. **目前的做法有哪些局限性？**
3. **您的方法有什么独特性，为什么您认为它会成功？**
4. **谁关心该项目取得的成果？**
5. **如果您成功了，对该领域的推动作用是什么？**
6. **中期和期末，如何检查该项目计划成功与否？**

## 2. 各部分核心使命

| 部分 | 核心使命 | 评审人最想看到的 |
|------|---------|----------------|
| 摘要 | 全文的浓缩快照 | 问题清晰、方法明确、成果可量化 |
| 立项依据 | **为什么做** | 重大需求、存在问题、有解决思路 |
| 研究内容 | **做什么** | 与科学问题的严格对应 |
| 研究目标 | **做到什么程度** | 可量化、可验证 |
| 研究方案 | **如何做** | 具体方法、可行路径、备选方案 |
| 可行性分析 | **凭什么能做** | 理论、条件、能力三维论证 |
| 创新点 | **与别人有什么不同** | 差异化明确、可验证 |
| 研究基础 | **我能做该项目** | 前期积累与本项目的直接关联 |

**目标效果**：让评审者读完后感觉——"这个研究很重要，国内外都在做，但有要害问题没有解决，申请人提出了很好的解决途径，思路很独特且合理，若沿着这条思路做几个方面的研究，有解决希望。"

---

## 3. 摘要结构（限400字符）

### 7句话模型

| 句序 | 功能 | 占比 |
|------|------|------|
| 第1句 | 研究背景：点出领域重要性 | ~15% |
| 第2句 | 研究现状：点出核心矛盾/空白 | ~15% |
| 第3-4句 | 科学问题与假说 | ~20% |
| 第5-6句 | 研究内容与方法（核心） | ~35% |
| 第7句 | 意义与价值 | ~15% |

### 关键要求

- 中文摘要 ≤ 400 字符，英文摘要 ≤ 4000 字符
- 关键词前置：前20字必须包含核心关键词
- 避免过度技术性术语（评审人可能非同一细分方向专家）

### 字数超限压缩策略

优先级：P0（核心论点）> P1（方法概述）> P2（背景修饰）。压缩顺序：先删修饰词 → 再合并相近表述 → 最后精简方法描述。最多3轮闭环压缩。

## 4. 立项依据结构

### 4.1 总体逻辑链

```
研究背景（为什么重要）
  → 国内外研究现状（别人做了什么）
    → 存在的问题与不足（还缺什么）
      → 科学假说/关键科学问题（我要怎么做）
        → 科学意义（做成了有什么价值）
```

### 4.2 推荐四段递进结构

**第一段：研究背景与科学意义**

四段递进式论证：
1. 开门见山，点出研究对象的重大意义（1-2句）
2. 解释核心术语/概念（2-3句）
3. 阐述研究价值和迫切性（1-2句）
4. 自然过渡到具体研究方向（1句）

**第二段：局限性与不足**

说明局限性和不足，引出本项目主要工作。包含：
- 风险转折（由优势过渡到问题）
- 案例引入（具体事例佐证）
- 场景风险（具体场景下的问题表现）

**第三段：解决途径**

介绍最新技术/方法如何有利于解决上述问题。段尾必须有总结性语言。

**第四段：项目贡献与意义**

说明本项目对领域的重要作用。意义总结应覆盖理论/技术/应用三维度。

### 4.3 国内外相关工作（分组三层结构）

每组相关工作遵循统一的三层结构：

**第一层（引入段）**：定义研究方向 + 说明重要性 + 划分3-4个子方向

**第二层（中间段）**：按子方向逐一展开，每个子方向包含：
1. 核心思路说明
2. 代表性工作列举（3-5项）
3. 指出不足

**第三层（综述段）**：总结进展 + 汇总缺陷

每段内部逻辑：`核心思路定义 → 工作列举 → 不足指出`

不足指出的三层递进：`承认进展 → 主体限制 → 能力缺失`

### 4.4 总结段

说明课题组具有解决上述挑战的能力：
- 综述项目定位和研究内容概览
- 展示团队在该领域的研究基础
- 点明项目的战略意义

### 4.5 创新点预告（立项依据末尾）

在立项依据末尾简要预告创新点，研究内容部分再详细展开。

### 4.6 约束与参数

| 参数 | 面上项目 | 青年基金 |
|------|---------|---------|
| 目标页数 | 5-7页 | 3-5页 |
| 目标字数 | 8000-10000 | 5000-7000 |
| 参考文献 | ≥30篇 | ≥25篇 |
| 近3年文献占比 | ≥50% | ≥50% |
| 自引比例上限 | ≤20% | ≤20% |
| 开篇300字 | 必须直击核心卡点 | 必须直击核心卡点 |

## 5. 研究内容结构

### 5.1 研究目标

- 3个以内目标，5-8行
- 用序号标出
- 用大同行能够理解的术语来描述

目标动词体系：

| 层次 | 动词 | 适用场景 |
|------|------|---------|
| 描述性 | 明确、确定、鉴定 | 现象/关系的确认 |
| 解释性 | 揭示、阐明、解析 | 机制/规律的探索 |
| 验证性 | 验证、证实、检验 | 假说/模型的验证 |
| 建设性 | 建立、构建、发展 | 方法/模型的创建 |
| 认知性 | 理解、认识、探讨 | 理论框架的深化 |

### 5.2 总体思路

- 建议先画研究内容框架图
- 描述各研究方向的整体关系和递进逻辑
- 采用三层架构描述（理论层→方法层→应用层）
- 说明层间关系（"相互支撑、逐级递进"）

### 5.3 各子内容通用结构（7步）

1. **小标题**：`研究内容X：***的***研究`
2. **主题定义句**（可选）
3. 提出该子内容要回答的具体问题（1-2句）
4. 说明研究方案/实验设计（3-5句）
5. 关键技术说明（如有创新方法需详述）
6. 可行性融入（1-2句）
7. 预期结果与判断标准（1-2句）

### 5.4 两阶段写作法（推荐）

**阶段1（规划）**：产出结构骨架（各子内容的标题、核心问题、方法关键词），请用户确认后再进入阶段2。

**阶段2（叙事）**：在确认的骨架上填充完整叙事，保持递进感和前后关联。

### 5.5 关键科学问题

论证模式：`现状描述 → 现有方法的不足 → "因此，如何..." → "是本项目拟解决的第X个关键科学问题"`

多个问题应形成递进链条（如：基础建模→过程推演→应对机制→溯源追踪）。

## 6. 研究方案结构

### 6.1 技术路线

- 先厘清框架图，再写文字
- 框架图通常比研究内容图更饱满
- 图文严格对应

### 6.2 层次编号体系

- 主标题：4.2.1、4.2.2（大方向）
- 一级子标题：（1）（2）（3）（维度）
- 二级子标题：①②③（具体技术路线）

### 6.3 可行性分析写法

**融入式（青年/面上基金推荐）**：在各子内容的方案描述后附1-2句可行性说明。

**独立成节式（重点/重大项目推荐）**：按三维度展开论证：

| 维度 | 论证对象 | 核心逻辑 |
|------|---------|---------|
| 方案可行 | 研究路线 | 理论成熟 + 技术已验证 + 符合产业方向 |
| 条件可行 | 机构/平台 | 拥有国家级平台 + 有相关基础设施 |
| 能力可行 | 人员团队 | 学术积累 + 项目经验 + 队伍优势互补 |

### 6.4 公式与文字协调（理工科）

- 公式前有推导说明
- 公式后有符号解释
- 不孤立呈现
- 递进模式：基础定义式 → 演化约束式 → 最终优化式

## 7. 创新点结构

### 7.1 推荐写法

特色与创新分开写：
- **特色**：长期积累的学术优势和技术优势
- **创新**：理论创新 / 方法创新 / 视角创新（三选二或三选三）

每个创新点的结构：`区别于现有… → 创新性地… → 有望突破…`

### 7.2 禁用词与推荐替换

| 禁用表述 | 原因 | 替换建议 |
|---------|------|---------|
| 填补国内/国际空白 | 暗示跟踪性研究 | 率先从…角度探索 |
| 首次发现 | 过于绝对，评审反感 | 创新性地提出 |
| 开创性 | 过于绝对 | 区别于现有研究 |
| 国内外尚无报道 | 无法证实 | 截至本申请提交时未见相关报道 |
| 本研究将彻底解决 | 不现实 | 有望为…提供新的解决思路 |
| 非常重要/极其关键 | 空泛修辞 | 用数据/事实说明重要性 |

## 8. 研究基础与工作条件结构

### 8.1 三个评审核心问题

研究基础部分需要回答三个核心问题：
1. **做过吗？** 前期成果与本项目的直接关联
2. **能做吗？** 技术储备和实验条件
3. **风险备案？** 可能的风险和应对方案

### 8.2 证据链优先级

直接相关成果 > 方法储备 > 平台条件 > 团队履历

### 8.3 结构

1. **总述**：在该领域的长期积累（1-2句）
2. **逐条前期成果**：每条配图表、关联研究内容（3-5条）
3. **风险应对措施**（2026年新增要求）：2-3个风险 + 具体应对方案
4. **工作条件**：已具备条件 + 尚缺条件 + 解决途径
5. **总结**：以上基础足以支撑本项目（1句）

### 8.4 风险应对模板

每个风险包含：
- **风险描述**：具体的失败场景
- **应对方案**：替代技术/备用资源/优化策略
- 风险数量：2-3个

## 9. 年度研究计划结构

### 9.1 与研究内容对应

每个年度的任务必须与研究内容条目对应，时间安排合理（不能把核心工作都放在第一年）。

### 9.2 三年期（青年基金）

- 第1年：完成基础体系搭建 + 研究内容一主体
- 第2年：完成核心实验 + 研究内容二和三主体
- 第3年：补充验证 + 数据整理 + 撰写论文 + 结题

### 9.3 四年期（面上项目）

- 第1年：完成研究内容1（基础方法），初步开展研究内容2
- 第2年：完成研究内容2（核心技术），初步开展研究内容3
- 第3年：完成研究内容3（应用验证），整理阶段成果
- 第4年：补充实验，撰写论文，结题

### 9.4 禁写内容

"购买试剂""查阅文献""学习…技术"等无意义内容。

## 10. 文献引用规范

### 10.1 引文要求

| 参数 | 面上项目 | 青年基金 |
|------|---------|---------|
| 总数 | 40-60篇 | 30-50篇 |
| 近3年占比 | ≥40% | ≥40% |
| 自引比例 | ≤20% | ≤20% |
| 中文文献 | 必须包含 | 必须包含 |
| 自己代表作 | 中英文各至少1篇 | 至少1篇 |

### 10.2 引文禁区

| 禁区 | 说明 |
|------|------|
| 自引超过20% | 显得自说自话 |
| 全部为英文文献 | 缺少对国内同行的关注 |
| 文献过旧（>10年为主） | 显得不了解前沿 |
| 引用教材/百科 | 显得不专业 |
| 引用未发表的论文 | 不可验证 |

### 10.3 引文写法要求

- 在正文中被**实质性引用**（说明该工作做了什么、有什么局限）
- 自己的代表作姓名**加粗**标注
- 分组介绍，每组介绍后简单总结

## 11. 选题策略

- 题目是对评审专家说的**第一句话**
- 题目应回答：**干什么、对象是什么、用什么方法、解决什么问题**
- 简洁明确，不宜过长
- 最好有新意的关键词（研究视角、方法和研究对象的创新）
- 以**问题为导向**选题，不要以新技术应用为导向
- 忌盲目追求"学科前沿"和"研究热点"
- 建议检索类似课题历年资助情况，避免重复

---

## 12. 核心写作策略

1. **完整闭环论证**：问题→分析→应对→意义
2. **多层次权威支撑**：国家政策、国际标准、行业实践多维度佐证
3. **从宏观到微观逐层深化**：全球趋势→国内政策→行业实践→具体技术难点→项目方案
4. **段落首句即主旨**：每段开头一句话即表明核心目的，便于快速浏览
5. **层层递进的编号体系**：大标题→（1）→①，确保读者快速定位层级
6. **重点内容标粗**：让专家即使只阅读加粗字体，也可获得判断本子优劣的关键信息
