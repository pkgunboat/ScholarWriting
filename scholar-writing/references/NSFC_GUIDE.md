# 国家自然科学基金申报书写作指南

本指南提供国自然基金申报书（面上项目和青年基金）的结构要求、评审侧重点、常见扣分项和写作技巧，供 Reviewer Agent 审稿时参考。

---

## 1. 面上项目结构要求

### 1.1 总体规范

- **正文总页数**：≤ 20 页（严格限制，超页直接影响评审印象）
- **字体与排版**：按系统模板，正文一般为宋体小四号，行距 1.25 倍

### 1.2 各部分建议篇幅

| 部分 | 建议页数 | 占比 | 说明 |
|------|---------|------|------|
| 摘要 | — | — | 300-500 字，独立于正文页数 |
| 立项依据与研究内容 | — | — | 以下细分 |
| ├─ 立项依据 | 3-5 页 | 25-35% | 含文献综述，是说服评审人的关键 |
| ├─ 研究内容 | 2-3 页 | 10-15% | 与科学问题严格对应 |
| ├─ 研究方案/技术路线 | 4-6 页 | 30-40% | 含技术路线图，是体现深度的核心 |
| ├─ 可行性分析 | 1-2 页 | 5-10% | 论证方案的可执行性 |
| └─ 创新点 | 0.5-1 页 | 3-5% | 精练、有实质内容 |
| 研究基础 | 2-3 页 | 10-15% | 前期工作与本项目的关联 |

### 1.3 摘要要求

- 字数：300-500 字
- 内容：研究背景（1-2 句）→ 科学问题（1 句）→ 研究内容（2-3 句）→ 预期成果（1 句）
- 关键词：3-5 个，中英文对照

---

## 2. 评审侧重点

评审人在有限时间内（通常 30-60 分钟审一本）重点关注以下方面：

### 2.1 科学问题的清晰度与新颖性

- 科学问题是否有明确的表述（而非笼统的"研究 X"）
- 是否区分了"技术问题"和"科学问题"
- 问题是否有足够的创新空间（既不是已解决的问题，也不是过于前沿无法验证的问题）

### 2.2 研究内容与科学问题的对应关系

- 每项研究内容是否都在回答科学问题的某个方面
- 研究内容之间是否有逻辑关系（递进、互补、并行验证）
- 是否有研究内容与科学问题无关（常见扣分项）

### 2.3 技术路线的可行性与具体性

- 是否给出了具体的方法名称、模型架构、评估指标
- 是否考虑了可能遇到的技术困难和备选方案
- 技术路线图是否与文字描述一致
- 关键步骤是否有充分的技术细节

### 2.4 前期积累

- 申请人是否在相关方向有已发表论文
- 前期工作与本项目的关联是否清晰
- 是否有初步的实验结果支撑可行性

### 2.5 创新点的实质性

- 创新点是否有实质性的技术贡献（而非仅仅是"首次将 A 用于 B"）
- 是否有与现有方法的具体对比来体现创新
- 创新点数量是否合理（2-3 个为宜，过多则不聚焦）

### 2.6 全文叙事的连贯性

- 从立项依据到研究内容到技术路线，是否有清晰的逻辑主线
- 各部分之间是否存在"断裂感"或"拼凑感"
- 文字风格是否统一（避免不同部分由不同人写导致的风格差异）

## 3. 常见扣分项

### 3.1 立项依据部分

- **综述过于宽泛**：花大量篇幅介绍领域背景，却没有聚焦到具体科学问题
- **文献综述不深入**：仅列出文献而未进行实质性分析和比较
- **科学问题表述模糊**：如"研究 GUI Agent 的关键技术"（哪些关键技术？什么科学问题？）
- **从综述到科学问题的过渡不自然**：综述了很多方向，但科学问题与综述关联不紧密

### 3.2 研究内容部分

- **内容过多**：超过 5 个研究内容，显得贪多求全、不聚焦
- **内容过于笼统**：如"研究高效的界面理解方法"（什么方法？怎样算高效？）
- **内容之间逻辑关系不清**：5 个研究内容各自独立，看不出整体架构
- **与科学问题脱节**：立项依据提出的问题没有在研究内容中得到回应

### 3.3 技术路线部分

- **缺乏具体性**：没有具体到方法名、模型架构、训练策略
- **评估方案缺失**：未说明用什么数据集、什么指标进行评估
- **技术路线图与文字不一致**：图中有的步骤文字没提到，或反之
- **缺少备选方案**：对可能遇到的技术困难没有预案

### 3.4 创新点部分

- **空洞化表述**：如"提出了一种新颖的方法"（新颖在哪里？）
- **缺乏对比**：没有说明与现有方法的具体区别
- **过度包装**：将常规技术手段包装成创新点

### 3.5 研究基础部分

- **与本项目关联不清**：列出了很多成果，但看不出与本项目的关系
- **前期积累不足**：在相关方向没有已发表论文或初步实验

### 3.6 格式与细节

- **参考文献格式不统一**
- **图表编号错误或缺失**
- **引文在正文中没有被引用**
- **年度计划与研究内容不对应**

## 4. 青年基金 vs 面上项目差异

| 维度 | 青年基金 | 面上项目 |
|------|---------|---------|
| 侧重点 | 申请人的成长潜力 | 研究方案的深度和系统性 |
| 研究内容数量 | 2-3 个即可 | 3-5 个 |
| 前期积累要求 | 有相关方向的基础即可 | 需要在该方向有深厚积累 |
| 创新点 | 可以偏探索性 | 需要有明确的技术路径 |
| 经费规模 | 20-30 万 | 50-80 万（信息领域） |
| 执行周期 | 3 年 | 4 年 |
| 评审标准 | 更看重潜力和方向选择 | 更看重方案的完整性和可行性 |

### 4.1 青年基金写作要点

- 突出个人学术成长轨迹，说明为什么选择这个方向
- 研究内容可以聚焦于一个核心问题的 2-3 个方面
- 前期积累不必面面俱到，但需要证明有能力独立开展研究
- 可以适当体现探索性和前瞻性

### 4.2 面上项目写作要点

- 强调研究方案的系统性，各研究内容之间要形成有机整体
- 技术路线要更加具体和详细，体现申请人的深厚功底
- 前期积累需要直接支撑本项目的可行性
- 创新点要有足够的技术深度，不能只是"组合创新"

## 5. 写作技巧

### 5.1 "漏斗式"立项依据

从大领域逐步收窄到具体问题，形成清晰的逻辑链条：

```
第 1 段：大领域的重要性和发展现状（1-2 段）
    ↓
第 2 段：该领域中一个具体方向的关键挑战（1-2 段）
    ↓
第 3 段：现有方法在应对该挑战时的不足（2-3 段，文献综述）
    ↓
第 4 段：不足背后的根本原因（科学问题的引出）（1 段）
    ↓
第 5 段：本项目的科学问题和研究思路（1 段）
```

每一层的过渡都应自然，让评审人感到"顺理成章"。

### 5.2 研究内容的"递进感"

研究内容的组织应体现逻辑递进，而非简单并列：

- **层次 1（基础）**：解决核心理论或方法问题
- **层次 2（核心）**：在层次 1 基础上构建核心技术
- **层次 3（应用/验证）**：将核心技术应用于具体场景并验证

各层次之间的输入输出关系要明确，让评审人看到"环环相扣"。

### 5.3 技术路线图的撰写

- 技术路线图应与文字描述严格对应
- 图中标注关键输入、输出、方法名称和评估指标
- 使用箭头表示数据流或逻辑依赖关系
- 图的粒度与文字描述一致（不要图比文字更详细或更粗糙）

### 5.4 年度计划与研究内容的对应

- 每个年度的任务应与研究内容条目对应
- 时间安排要合理（不能把所有核心工作都放在第一年）
- 建议格式：

```
第 1 年：完成研究内容 1（基础方法），初步开展研究内容 2
第 2 年：完成研究内容 2（核心技术），初步开展研究内容 3
第 3 年：完成研究内容 3（应用验证），整理成果
第 4 年：补充实验，撰写论文，结题
```

### 5.5 参考文献的使用

- 参考文献要在正文中被实质性引用（不只是列在最后）
- 引用时要体现"读过"：说明该工作做了什么、有什么局限
- 引文数量：面上项目建议 40-60 篇，青年基金 30-50 篇
- 近 3 年的文献应占 40% 以上，体现对领域前沿的跟踪
- 适当引用申请人自己的相关工作，建立前期积累的说服力
- 避免引用过多综述文章，以原始研究论文为主

### 5.6 摘要的写作

摘要是评审人阅读的第一部分，决定了第一印象：

- 第 1-2 句：研究背景和科学问题
- 第 3-4 句：研究内容概述（与正文研究内容条目对应）
- 第 5 句：技术路线的核心思路
- 第 6 句：预期成果和意义

避免在摘要中使用过于技术性的术语（评审人可能不是同一细分方向的专家）。
