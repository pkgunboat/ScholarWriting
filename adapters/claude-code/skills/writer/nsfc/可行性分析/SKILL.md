# 可行性分析 Writer

## 任务

撰写国自然申报书的「可行性分析」部分。回答评审人最关心的问题："凭什么你能做成？"

你将收到：
1. 该章节的大纲（来自 planning/outline.md）
2. 该章节的评审标准 checklist（写作时请对照）
3. 已完成前序章节的摘要（200-300 字/章）
4. 相关素材片段（来自 planning/material_mapping.md）
5. 论点注册表（planning/claim_registry.md）
6. 研究方案全文（可行性必须针对方案中的具体技术环节论证）

## 写作策略

### 三个评审核心问题

可行性分析的本质是回答三个问题：
1. **做过吗？** 前期积累与本项目的直接关联
2. **能做吗？** 技术储备和实验条件是否满足
3. **风险备案？** 关键环节失败时有无替代方案

### 写法选择

根据项目类型选择不同的写法：

**融入式（青年/面上基金推荐）**：可行性说明融入研究方案各子内容中（每个方案后附1-2句），本章节仅做简短汇总（1-2页）。

**独立成节式（重点/重大项目推荐）**：在本章节按三维度完整展开论证。

### 三维度论证框架

#### 维度1：方案可行性

论证研究路线在理论和技术上的可行性：

- 核心方法的理论成熟度（是否有成熟的理论基础支撑）
- 关键技术的已验证性（是否在相关场景中被验证过）
- 方案整体的逻辑自洽性（各环节是否层层递进、互相支撑）
- 与主流技术路线的一致性（是否符合产业和学术发展方向）

**论证方式**：每个关键技术环节列出支撑证据（前人工作[ref]、本组前期成果、业界实践）。

#### 维度2：条件可行性

论证实验条件和平台支撑：

- 依托单位的相关平台（国家重点实验室、省部级平台等）
- 已建成的关键设施（计算集群、实验设备、数据集等）
- 合作资源（与外部单位的合作协议、共享平台等）
- 与本项目研究内容的匹配度

**注意**：如果有基于国产人工智能生态的开源基础，建议重点突出。

#### 维度3：能力可行性

论证团队有足够的能力执行方案：

- 申请人在相关方向的学术积累（发表论文、承担项目）
- 团队成员的互补优势（不同成员覆盖不同技术环节）
- 研究生培养基础
- 与本项目所需能力的匹配度

### 与研究基础的差异

可行性分析和研究基础都涉及前期积累，但侧重不同：

| 维度 | 可行性分析 | 研究基础 |
|------|----------|---------|
| 核心问题 | 凭什么能做成？ | 做过什么相关工作？ |
| 侧重点 | 能力和条件 | 成果和积累 |
| 论证方向 | 前瞻性（证明未来可行） | 回顾性（展示过去成就） |
| 粒度 | 针对方案中的具体技术环节 | 按研究方向综述 |

### 字数与篇幅

- 面上项目：1500-3000字（1-2页）
- 青年基金：1000-2000字（1页）

### 质量要求

1. **具体证据**：每个维度都要有具体证据（论文引用、代码链接、设备型号、项目编号等），不能空谈
2. **与方案对应**：方案中的每个关键技术环节至少有一个可行性支撑点
3. **避免自吹自擂**：用事实说话，避免"国际领先""国内首创"等空洞表述
4. **避免与研究基础重复**：引用相同成果时，侧重不同角度
