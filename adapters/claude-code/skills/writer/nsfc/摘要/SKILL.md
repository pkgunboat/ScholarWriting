# 摘要 Writer

## 任务

撰写国自然申报书的「摘要」部分。摘要是评审人阅读的第一部分，决定第一印象。

你将收到：
1. 该章节的大纲（来自 planning/outline.md）
2. 该章节的评审标准 checklist（写作时请对照）
3. **所有已完成章节的全文**（摘要最后写，依赖所有其他章节）
4. 相关素材片段（来自 planning/material_mapping.md）
5. 论点注册表（planning/claim_registry.md）

## 写作策略

### 核心原则

摘要是全文的浓缩快照，不是重写。直接从各章节提取核心句，压缩为紧凑的整体。

### 7句话模型

| 句序 | 功能 | 占比 | 提取来源 |
|------|------|------|---------|
| 第1句 | 研究背景：点出领域重要性 | ~15% | 立项依据第一段 |
| 第2句 | 研究现状：点出核心矛盾/空白 | ~15% | 立项依据的研究空白段 |
| 第3-4句 | 科学问题与假说 | ~20% | 立项依据的科学问题段 |
| 第5-6句 | 研究内容与方法（核心，最大篇幅） | ~35% | 研究内容 + 研究方案 |
| 第7句 | 意义与价值 | ~15% | 创新点 |

### 写作流程

**步骤1：提取核心句**

从已完成章节中提取每个句位的候选表述：
- 第1句：从立项依据开篇提取研究对象的核心定位
- 第2句：从立项依据的研究空白段提取最关键的矛盾/不足
- 第3-4句：从科学问题/假说段提取核心问题和假说表述
- 第5-6句：从研究内容的各子内容中各取一句核心方法描述，压缩组合
- 第7句：从创新点中提取最核心的贡献表述

**步骤2：组合与压缩**

将提取的核心句组合为连贯段落。检查总字符数：
- 如果 ≤ 400字符：完成
- 如果 > 400字符：进入压缩流程

**步骤3：闭环压缩（如需）**

压缩优先级（先删低优先级内容）：
- P2（低优先级）：背景修饰词、领域重要性的泛泛表述
- P1（中优先级）：方法描述中的技术细节、评估指标
- P0（高优先级，最后才动）：核心论点、科学问题表述、关键方法名

压缩操作顺序：先删修饰词 → 再合并相近表述 → 最后精简方法描述。每轮压缩后重新计数，最多3轮。

### 英文摘要

中文摘要定稿后，翻译为英文摘要：
- 英文摘要 ≤ 4000字符
- 忠实翻译，不增删内容
- 使用标准学术英语，避免直译

### 关键词

- 3-5个关键词，中英文对照
- 第一个关键词应为研究领域
- 最后一个关键词可以是方法论或应用领域

## 质量要求

1. **关键词前置**：前20个字符内包含核心术语
2. **信息密度**：每句话都有信息量，不留空话
3. **避免技术黑话**：评审人可能不是同一细分方向专家，用大同行能理解的术语
4. **与正文一致**：所有表述必须与正文各章节一致，不引入新概念
5. **使用中文标点**：全角标点符号

## 输出

将完成的章节写入 `sections/01_摘要.md`，包含：
- 中文摘要（≤400字符）
- 关键词（3-5个）
- 英文摘要（≤4000字符）
- Keywords（与中文关键词对应）

## 约束

1. 引用 claim_registry 中的论点表述，与其他章节保持一致
2. 遵循 `references/STYLE_GUIDE_ZH.md` 中的中文学术写作规范
3. 不捏造数据、引文或实验结果——所有数据必须来自用户提供的素材
4. 避免 AI 写作痕迹（参考 `references/DEAI_PATTERNS_ZH.md`）
5. 写作时对照 checklist，确保覆盖所有 critical 和 high 级别的标准
6. 句式参考 `references/patterns/00_通用.md` + `references/patterns/01_摘要.md`
7. 结构参考 `references/NSFC_STRUCTURE_ZH.md` 第3节
